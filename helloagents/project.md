# 项目技术约定

> 这是一份“写给自己看的项目说明书”。遇到文档和代码不一致时，以代码为准，并把文档更新回来。

---

## 技术栈（计划）

- **运行环境:** Node.js（当前开发机版本即可）
- **前端/后端一体:** Next.js（TypeScript）
- **UI 样式:** Tailwind CSS（为了快速复用原型样式）
- **图标:** Font Awesome（当前用 CDN，后续可替换为本地依赖）
- **搜索来源:** Tavily（全网搜索 API）
- **大模型:** OpenAI（用于把搜索结果提炼成“候选游戏 + 证据 + 打分理由”）
- **测试:** Vitest（最小单元测试）

---

## 开发约定（大白话版）

- **API key 只放后端:** 任何 `API key` 都只能放在本地环境变量（`.env.local`）里，禁止写进前端代码、禁止提交到 git。
- **先做 Demo 可用，再优化:** 先把“能搜→能排→能解释→能动画展示”跑通，再慢慢提高准确率与美观度。
- **每一步都可解释:** 每次筛选/加分/减分都要给出“因为你说了什么/因为网页里出现了什么”这种可读理由。

---

## 常用命令

- 开发启动：`npm run dev`
- 构建检查：`npm run build`
- 类型检查（我们把它叫 lint）：`npm run lint`
- 单元测试：`npm test`

---

## 环境变量（计划）

这些变量会写在本地的 `.env.local`（不提交）：

- `TAVILY_API_KEY`: Tavily 的 key（用于全网搜索）
- `OPENAI_API_KEY`: OpenAI 的 key（用于信息提炼与打分解释）

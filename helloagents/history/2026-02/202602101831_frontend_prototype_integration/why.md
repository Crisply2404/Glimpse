# 变更提案: 接入 AIStudio 前端原型（Tailwind + 事件流播放）

## 需求背景
目前项目已经有后端接口（`/api/recall`）能返回“候选 + 证据 + 过程事件流”，但前端页面还是一个基础 Demo。你已经用 AIStudio 做好了更符合设想的前端原型（含：输入区/过程可视化/扭蛋机/结果卡片）。

这次变更的目标就是：把这个原型“接进来”，并且真正对接我们的 `/api/recall`，让它从“演示用假数据”变成“真实跑起来”。

## 产品分析

### 目标用户与场景
- **用户群体:** 想靠零碎记忆找回一款游戏的人
- **使用场景:** 输入模糊描述 → 看见候选逐步被筛掉 → 扭蛋机依次掉出 Top 结果 → 点开证据链接验证

### 价值主张与成功指标
- **价值主张:** “不只给答案，还把筛选过程演出来”
- **成功指标（本次迭代）:**
  - AIStudio 原型能在 Next.js 里正常显示（外观基本一致）
  - 点击 Start 后会调用 `/api/recall`，并按返回的 `events` 做“过程回放”
  - Top5 结果可逐个“扭蛋掉落”，每个结果能查看证据链接

### 人文关怀
- 文案尽量用大白话，少用专业词
- 不收集用户隐私，Demo 阶段不做账号

## 变更内容
1. 在 Next.js 项目中加入 Tailwind（原型大量使用 Tailwind class）
2. 迁移并适配 AIStudio 的组件结构（Header/InputArea/Visualizer/Results/GachaMachine）
3. 把“mock 流程”替换为真实 API：`POST /api/recall`
4. 保持 `API key` 只在服务端环境变量中使用

## 影响范围
- **模块:** 前端（可视化）
- **文件:** 多文件新增/替换（Tailwind 配置 + 组件迁移）
- **API:** 不变（继续使用 `/api/recall`）
- **数据:** 不落库（仍为请求内数据 + 内存缓存）

## 核心场景

### 需求: 原型 UI 能跑起来并对接真实数据
**模块:** 前端（可视化）

#### 场景: Start 后播放事件流
用户点击 Start 后，前端调用 `/api/recall` 获取结果，然后按 `events` 逐条播放。
- 预期结果: 时间线出现“搜索/提炼/筛选/打分/扭蛋”等步骤；候选池随筛选变灰/淘汰

### 需求: 扭蛋机逐个掉出 Top5
**模块:** 前端（可视化）

#### 场景: 逐个 Reveal
到达扭蛋阶段后，扭蛋机自动/点击逐个掉出 Top5。
- 预期结果: 每掉出一个，就显示对应结果卡片（含分数与证据链接）

## 风险评估
- **风险:** Tailwind 配置不一致导致原型样式跑偏
- **缓解:** 采用 Tailwind darkMode=class；补齐原型用到的自定义动画/滚动条样式
